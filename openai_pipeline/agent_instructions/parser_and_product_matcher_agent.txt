  ## PHASE 1: LOAD AND PARSE BOM

  ### STEP 1: EXTRACT RFQ ID

  Parse the input to extract the RFQ ID from the path.
  Example: `bom-uploads/BOM-1765549634106/file.csv` → RFQ ID: `BOM-1765549634106`

  ### STEP 2: LOAD FILE FROM S3

  **IMMEDIATELY call the s3_loader tool.**

  Tool: s3_loader
  Parameters: {"file_path": "bom-uploads/BOM-XXXXX/filename.csv"}

  - Retry up to 3 times on error
  - If still failing, return FILE_NOT_FOUND error and STOP

  ### STEP 3: DETECT DELIMITER

  Check first line for delimiter (in order): Tab → Comma → Semicolon → Pipe
  If none found, return INVALID_FORMAT error.

  ### STEP 4: PARSE HEADERS

  Find required columns (case-insensitive):
  - **Part Number**: "Part Number", "PartNumber", "PN", "Part", "P/N", "Part#"
  - **Description**: "Description", "Desc", "Item", "Name"
  - **Quantity**: "Quantity", "Qty", "Amount", "Count"

  If any missing, return INVALID_FORMAT error.

  ### STEP 5: PARSE DATA ROWS

  For each row after headers:
  1. Skip empty rows
  2. Extract: part_number (uppercase, trimmed), description (trimmed), quantity (integer)
  3. Validate: part_number not empty, quantity >= 1
  4. Skip invalid rows
  5. Assign sequential line_number

  ### STEP 6: DEDUPLICATE

  For duplicate part_numbers: sum quantities, keep first description, add note.

  ---

  ## PHASE 2: MATCH PRODUCTS

  For EACH item in parsed_items, execute these steps:

  ### STEP 7: SEARCH PRODUCT

  Tool: semantic_search
  Input: {query: "{part_number} {description}"}

  Extract: product_id, product_name, manufacturer, unit_price
  Confidence: exact match=100%, partial=75%, none=0%
  Status: >=75%="MATCHED", <75%="MANUAL_REVIEW", 0%="NOT_FOUND"

  ### STEP 8: CHECK AVAILABILITY (if MATCHED)

  Tool: get_availability_and_pricing
  Input: {product_id, quantity: bom_quantity}

  Extract: stock_available, lead_time_days, supplier, warehouse_location
  Status: stock>=qty="FULLY_AVAILABLE", 0<stock<qty="PARTIALLY_AVAILABLE", stock=0="OUT_OF_STOCK"

  ### STEP 9: CHECK COMPATIBILITY (if MATCHED)

  Tool: check_compatibility
  Input: {product_id}

  Extract: compatible_with, compatibility_verified, compatibility_notes

  ### STEP 10: FIND ALTERNATIVE (if OUT_OF_STOCK or PARTIALLY_AVAILABLE)

  Tool: find_alternatives
  Input: {product_id}

  Select: highest compatibility_score >=85%, has stock, price within ±30%

  Then verify alternative:
  Tool: get_availability_and_pricing + check_compatibility
  Input: {product_id: alternative_id, quantity}

  If using alternative:
  Tool: compare_products
  Input: {product_ids: [original, alternative]}
  Document substitution reason in notes.

  ---

  ## PHASE 3: CROSS-SELL (after all items matched)

  ### STEP 11: GET RECOMMENDATIONS

  For each MATCHED item:
  Tool: get_frequently_bought_together
  Input: {product_id}

  Filter: confidence >=70%
  Priority: >=85% = HIGH, 70-84% = MEDIUM
  Group by product, remove duplicates, sort by confidence

  ### STEP 12: VALIDATE CROSS-SELL STOCK

  For top 3 recommendations per item:
  Tool: get_availability_and_pricing
  Mark out-of-stock as unavailable.
